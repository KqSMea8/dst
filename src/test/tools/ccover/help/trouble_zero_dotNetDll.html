Measurements Show No Coverage for .NET DLL

SYMPTOMS

BullseyeCoverage shows 0% coverage or less than actual coverage for the functions in a DLL built with Microsoft Visual C++ .NET 2003 using the Common Language Runtime (cl option -clr).

CAUSE

The Common Language Runtime prevents the DLL entry point call with argument DLL_PROCESS_DETACH. BullseyeCoverage normally intercepts this call to save coverage from memory to file.

RESOLUTION

Add a call to cov_write after the last executable statement in the DLL.

MORE INFORMATION

The BullseyeCoverage auto-save background thread might save some or all coverage measurements before the DLL is unloaded.

The DLL entry point function is documented by Microsoft under the name DllMain.

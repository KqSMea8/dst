Green Hills MULTI

This product is not officially supported as of September 2009. Although we will make an effort to resolve issues with this product, we cannot guarantee a successful outcome.

The Green Hills MULTI IDE cannot be configured to build using BullseyeCoverage. However, you can build MULTI projects from a command prompt. This documentation applies to MULTI version 4.2.

Build Run-Time Library

Use the commands below to build the BullseyeCoverage run-time library. Modify the compile command to use the same compile options as your project. 

cd BullseyeCoverage/run
ccintppc -os_dir /GHS/int5011/ -bsp sim800 -c libcov-greenHills-integrity.c
ax -r -c ../lib/libcov-greenHills-integrity.a libcov-greenHills-integrity.o

Build Project with a Script

  1. Use the Green Hills gbuild program to generate a script from your MULTI project as shown below. 
    cd projectDirectory
gbuild -allinfo -commands -unix >build.sh

  2. Modify build.sh so that it invokes the compiler without an absolute path. For example, change: 
    greenHillsDirectory/gcx -c -MD -bsp generic -G -object_dir=objs --arm \

    to 

    gcx -c -MD -bsp generic -G -object_dir=objs --arm \

  3. Add the BullseyeCoverage/bin directory to the front of the PATH environment variable.

  4. Run the script. If you are working on Windows, use a Unix-style shell to run this script, due to line continuations generated by gbuild. A shell that works well is available at www.bullseye.com/pub/. 
    cov01 -1
sh -e build.sh

Running on INTEGRITY

The BullseyeCoverage run-time first attempts to open files using host I/O (hostio_open). If that fails, the run-time attempts to open files on the native file system.

The BullseyeCoverage run-time reads environment variables from the file \BullseyeCoverageEnv.txt. This ASCII text file has the format below: 

name1=value1
name2=value2
...

Syntax Error in Template Function

SYMPTOMS

BullseyeCoverage reports a syntax error inside a template function that is not instantiated. No such error occurs when compiling without BullseyeCoverage.

CAUSE

The function contains a syntax error. The error is not reported without BullseyeCoverage because the compiler does not check syntax of template functions unless they are instantiated. BullseyeCoverage checks syntax regardless of instantiation.

RESOLUTION

Fix the syntax error.

MORE INFORMATION

These compilers defer syntax checking until instantiation.

  • Borland C++
  • CodeWarrior
  • GCC 3.3 and earlier
  • HP aC++ 6
  • IBM C++
  • Intel C++
  • Microsoft C++
  • SGI MIPSpro
  • Sun C++

These compilers do not defer syntax checking.

  • GCC 3.4 and later
  • HP aC++ 3

You can demonstration the compiler behavior with the source code below. The source contains an error that is not reported. Uncomment the line in main to see the error reported. 

template<typename T>
void f()
{
    error
}

int main()
{
    //f<int>();
    return 0;
}

Locating Files

This section describes how you inform BullseyeCoverage of the location of the coverage file as well as the location of the source files.

BullseyeCoverage requires source files only for building and for the annotated source code report.

BullseyeCoverage stores coverage data in one file, named test.cov by default. The covc command creates the coverage file if it does not exist and adds an entry for each source file processed. When you run a test program, the BullseyeCoverage run-time library reads and writes the coverage file. The coverage file does not grow in size when the test program runs.

You can specify an alternate directory or filename for the coverage file with the environment variable COVFILE or with the command line option -f (--file). If you have source files in multiple directories, it is very important that you specify the location of the coverage file; do not use the default. Since the BullseyeCoverage run-time uses the environment variable COVFILE and has no command line option to locate the coverage file, setting COVFILE is generally preferred over the command line option. Set COVFILE to the absolute path (fully qualified filename) of the coverage file.

The example commands below set COVFILE at the command prompt on various operating systems. 

set COVFILE=c:\source\test.cov (Windows)
export COVFILE=`pwd`/test.cov  (Unix Bourne/Korn shell)
setenv COVFILE `pwd`/test.cov  (Unix C shell)

On Windows you can set environment variables in the Control Panel under System. Be careful not to write spaces at the beginning or end of an environment variable setting.

covc stores source file names relative to the source directory. By default, the source directory is the directory where the coverage file is located. If you need to locate the coverage file in a directory other than where the source files are located, use environment variable COVSRCDIR or command line option -d (--srcdir) to specify the source directory. For example, consider a project consisting of the following source files. 

/project/src/dir1/source1.c
/project/src/dir1/source2.c
/project/src/dir2/dir3/source3.c
/project/src/source4.c

In the simplest case, you would set COVFILE to /prj/src/test.cov. But suppose you do not have permission to create the data file in /project/src. In this case, you might set COVFILE to /user/steve/test.cov and set COVSRCDIR to /project/src.

If you move source files or change the environment variable COVSRCDIR or option -d (--srcdir), delete the coverage file and rebuild your project.

If you do not specify the source directory explicitly, do not move the coverage file. Rather, delete the coverage file and rebuild your project.

If your run-time operating system does not support environment variables or does not provide the function getenv, you can set the value of the macro Libcov_covfile to a filename other than test.cov. For details, see Custom and Embedded Environments: Build Options.

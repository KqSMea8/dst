Coverage Calculation

The function coverage of a region is the ratio of the number of functions invoked to the total number of functions in the region.

The condition/decision coverage of a region is the ratio of the items listed below to the total possible within the region.

  • the number of decisions and conditions which evaluated true
  • the number of decisions and conditions which evaluated false
  • the number of case and default labels encountered
  • the number of C++ catch clauses encountered

The decision coverage of a region is the same as the condition/decision, except that condition outcomes are not counted.

A decision is the entire expression tested by a control structure. It may include logical operators. A condition is an expression separated by logical operators. In example below, there are two conditions, A and B, and one decision A || B. 

if (A || B) ...

BullseyeCoverage truncates fractional percentage numbers to the next lower percentage. For example, 999/1000 is shown as 99% rather than 100%. This way, "100%" only appears for fully covered regions.

The uncovered functions and uncovered conditions/decisions are simply the difference between the total number of items and the number invoked.

Introduction

What is BullseyeCoverage

BullseyeCoverage is a code coverage analyzer for programs written in the C and C++ languages. With BullseyeCoverage, you can:

  • identify code which was not fully exercised during testing
  • obtain summary measurements of testing completeness to help you decide when you have tested enough

The steps involved in using BullseyeCoverage are:

  1. Enable Coverage Build and build your program
  2. Run your program one or more times
  3. Generate and view coverage reports
  4. Repeat from step 2 until you are satisfied with testing

How it Works

When you compile code with BullseyeCoverage enabled, information about the code is written to a coverage file. The BullseyeCoverage run-time library collects coverage measurements in memory and periodically writes this data to the same coverage file. Environment variable COVFILE names the coverage file. You can view the coverage file using the Coverage Browser or the command line report programs.

Measurement Technique

BullseyeCoverage is an intrusive coverage analyzer; it adds probes to C and C++ source code. There is no measurement of assembly code. BullseyeCoverage measures condition/decision coverage and function coverage. A short description of these measures is given below. A detailed description of these measures, and others, is included in http://www.bullseye.com/coverage.html.

BullseyeCoverage measures condition/decision coverage by default. This measure reports the following information.

  1. whether each controlling expression in if, if-else, while, do-while, and for statements evaluates to both true and false
  2. whether each sub-expression separated by logical-and (&&) and logical-or (||), evaluates to both true and false.
  3. whether the controlling expression in each conditional-expression (?:) that does not occur inside another controlling expression evaluates to both true and false,
  4. whether each switch label is selected
  5. whether each function is invoked
  6. whether each C++ catch clause is entered
  7. whether control reached the end of each C++ try-block

Performance

Instrumenting with BullseyeCoverage typically increases execution time to 1.2x, code size to 1.4x and compile time to 1.7x with most Microsoft C++ projects, 3-8x with other compilers. Your results may vary. Compiler optimization settings greatly affect performance of instrumented code.

Typographical Conventions

Text in code font indicates text literally as it appears in a program or in a command. Text in italic font indicates a place holder for text that varies or that you supply. Text that appears within square brackets [ ] is optional.

The word BullseyeCoverage (italic) refers to the directory where you install BullseyeCoverage.

For Windows, we sometimes show the directory separator character as a forward slash / instead of the more conventional backslash \. This is for simplicity since BullseyeCoverage accepts the forward slash / as the directory separator on all operating systems.
